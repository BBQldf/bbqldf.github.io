# 文件架构规范（v1）

## 1. 目标
- 所有“项目（deliverable/supplement）”必须可追溯到明确文件。
- 所有 AI 输出必须可追溯到规则/事实来源文件。
- 目录结构稳定，避免功能散落与单文件膨胀。

## 2. 顶层目录规范
- `departments/`：部门产品与运行资产的统一真值源。
- `services/`：运行时服务与编排逻辑。
- `db/`：运行时 SQLite 数据。
- `schemas/`：协议与数据结构定义。
- `scripts/`：运维/生成/校验脚本。
- `docs/`：手册与架构文档。

## 3. 部门目录规范（每个 agent 必须一致）
以 `departments/{department_key}/` 为单位，必须包含：
- `agent.md`
- `blackboard.md`
- `role_description.md`
- `capability_description.md`
- `skills_extends/`
- `server.yaml`
- `rule.yaml`
- `product.yaml`

兼容项：
- `role_descrpiton.md` 仅作历史兼容，后续统一收敛到 `role_description.md`。

## 4. 服务层规范
- `services/department-service/src/`：部门服务通用实现（HTTP、DB、分析流水线）。
- `services/department-service/src/analysis/`：四层分析 + Q&A。
- `services/bigboss-service/src/server.py`：全局聚合与统一 API 入口。
- `services/orchestrator-service/web/`：BigBoss/部门前端页面。
- `departments/{department}/server.yaml`：部门独立部署配置。
- `services/bigboss-service/config/server.yaml`：BigBoss 独立部署配置。

## 5. 项目映射规范
- 项目唯一标识：
  - 职责输出：`deliverable_id`
  - 其他补充：`supplement_id`
- 源文件固定：`departments/{department}/blackboard.md`。
- 源字段路径固定：`deliverables[i]` 或 `supplements[i]`。
- 运行时存储固定：
  - `db/department_{agent}.db::fact_events(project_id=...)`
  - `db/department_{agent}.db::analysis_feedback(project_id=...)`

## 6. 生成与校验
- 生成项目映射：
  - `python3 scripts/generate_project_file_map.py`
- 校验架构：
  - `python3 scripts/validate_architecture.py`

## 7. 当前映射清单
- 机器可读总清单：`docs/architecture/PROJECT_FILE_MAP.yaml`
- 系统模块到文件映射：`docs/architecture/01_系统项目到文件映射.md`
